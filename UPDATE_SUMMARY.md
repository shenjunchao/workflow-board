# workflow-board 更新总结

版本：v1.0.0

## 本次更新重点

本次更新主要围绕工作流编辑体验、界面精简、节点内容编辑和部署可维护性进行优化。数据结构和 Supabase 表结构未变更。

## 功能更新

- 缩放范围调整：
  - 最小缩放改为 10%。
  - 最大缩放改为 300%。
- 工具栏精简：
  - 移除“自动布局”按钮。
  - 移除右上角“保存”按钮。
  - 移除右上角“恢复示例/刷新”按钮。
  - 保留自动保存逻辑，不影响本地和云端同步。
- 按钮位置调整：
  - “新增节点”移动到画布工具栏。
  - “新增节点”位于“新增连线”左侧。
- 节点内容编辑：
  - 节点右上角新增编辑图标。
  - 点击编辑图标后，当前节点进入编辑状态。
  - 编辑状态下可直接修改节点标题和备注内容。
  - 再次点击编辑图标退出编辑状态。
  - 编辑时不会触发节点拖拽。
- 版本号提示：
  - 左下角帮助图标悬停时显示版本号：`v1.0.0`。

## 界面优化

- 修复节点数量较多时，左侧列表把整个工作区撑高的问题。
- 左侧节点列表改为内部滚动。
- 工作区保持固定视口高度，画布不再被列表内容拉长。
- 节点内部排版优化：
  - 分类标签和标题之间增加间距。
  - 标题和正文内容之间增加间距。
  - 节点标题字号从 13px 提升到 17px。

## 稳定性优化

- 修复复制图片节点时可能复用旧图片路径的问题。
- 图片上传失败时保留本地图片状态，后续可继续重试同步。
- 保持 localStorage 本地优先保存，云端同步失败不会阻断本地使用。
- 修复 CSS 中异常的 `\n` 字符残留。
- 强化数据保存与云端同步：
  - 编辑后立即写入浏览器本地数据，避免防抖队列未执行导致内容丢失。
  - 云端保存改为串行执行，避免慢请求用旧数据覆盖新数据。
  - 页面切后台、刷新或关闭前会强制落盘，并尽力触发云端保存。
  - 登录同步时按项目逐个比较更新时间，避免一个旧项目覆盖另一个新项目。

4. 检查“新增节点”是否位于“新增连线”左侧。
5. 检查节点右上角编辑图标是否可进入/退出编辑模式。
6. 检查左下角帮助图标悬停是否显示版本号。
7. 检查已有项目数据是否正常加载。
