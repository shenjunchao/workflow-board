# workflow-board 更新总结

版本：v1.0.1

## 本次重点

本次修复重点是图片项目加载慢、图片加载不出来的问题。未修改 Supabase 表结构，不需要重新执行 `supabase-schema.sql`。

## 性能优化

- 图片签名 URL 改为批量获取，避免每张图片单独请求 Supabase。
- 图片节点先显示“图片加载中”占位，签名 URL 返回后只替换对应图片，不再触发整页重渲染。
- 图片元素增加 `loading="lazy"` 和 `decoding="async"`，减少首屏渲染压力。
- 签名 URL 增加前端缓存，重复进入项目时减少重复请求。
- 签名 URL 缓存增加过期控制，避免长时间打开页面后继续使用失效图片链接。
- 批量签名失败时保留单张签名兜底，避免单次批量失败导致所有图片不显示。
- 优化滚轮缩放和中键平移：
  - 滚轮缩放改为乘法比例缩放，手感更连续。
  - 缩放和平移过程只更新画布视口，不再重建节点 DOM、列表和详情面板。
  - 视口更新使用 `requestAnimationFrame` 合帧，保存操作延后到操作停止后执行。

## 稳定性修复

- 修复脚本中被编码破坏的未闭合字符串，`workflow-board.js` 已通过语法检查。
- 保持已有本地数据、云端数据和 Supabase 数据结构不变。
- 修复线上状态栏仍显示乱码的问题，清理云端同步、保存、退出、图片处理等底部提示文案。
- 为 `workflow-board.css`、`supabase-config.js`、`workflow-board.js` 增加 `v=1.0.2` 版本参数，避免 GitHub Pages 或浏览器继续加载旧缓存。
- 修复项目删除后又重新出现的问题：
  - 删除项目时同步切换当前活动项目，避免自动保存把旧项目重新写回本地。
  - 增加已删除项目记录，云端旧快照合并时不会把已删除项目恢复出来。
  - 删除后优先触发一次云端保存，减少多设备数据不一致。

## 上线后检查

1. 打开 GitHub Pages 页面后使用 `Ctrl + F5` 强制刷新。
2. 登录账号，进入包含图片的项目。
3. 确认节点先显示占位，然后图片逐步加载出来。
4. 换无痕窗口登录同一账号，确认图片仍能从云端加载。
