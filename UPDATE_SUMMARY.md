# workflow-board 更新总结

版本：v1.0.3

## 本次重点

本次更新彻底清理项目中的乱码问题，并增加旧数据自动修复逻辑。未修改 Supabase 表结构，不需要重新执行 `supabase-schema.sql`。

## 乱码修复

- 清理 `index.html` 和 `workflow-board.html` 中残留的静态乱码文案。
- 清理 `workflow-board.js` 中可见按钮、提示、弹窗、工具栏、状态栏、下拉菜单文案。
- 修复分类下拉框中旧乱码分类名的问题。
- 新增旧数据清洗逻辑，加载本地和云端数据时会自动修正常见乱码分类、标题和链接文案，并避免误清空正常中文内容。
- 清理代码注释中的乱码，避免后续误判。
- 将入口资源版本号提升到 `v=1.0.3`，避免 GitHub Pages 或浏览器继续使用旧缓存。

## 已保留能力

- 图片懒加载和批量签名 URL。
- 项目删除墓碑记录，避免删除后复活。
- 滚轮缩放和中键平移轻量渲染。
- 本地与 Supabase 云端同步。

## 需要上传到 GitHub 的文件

- `index.html`
- `workflow-board.html`
- `workflow-board.js`
- `UPDATE_SUMMARY.md`

## 上线后检查

1. 等 GitHub Pages 部署完成。
2. 打开页面后按 `Ctrl + F5` 强制刷新。
3. 进入项目，检查顶部工具栏、分类下拉框、右侧详情、底部状态栏是否无乱码。
4. 如果某个旧分类仍乱码，打开项目后保存一次或点击“一键上传云端”，让清洗后的数据覆盖云端。
